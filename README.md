# To-Do List Application


Полнофункциональное веб-приложение для управления повседневными задачами со сквозной системой приоритетов и гибкой клиентской фильтрацией. Данный этап разработки посвящен глубокому рефакторингу монолитной архитектуры бэкенда и повышению отзывчивости интерфейса.

## Ссылки проекта

*   **Репозиторий на GitHub:** https://github.com/AndreyGlitet/to-do-list 
*   **Рабочий деплой:** https://to-do-list-production-aa24.up.railway.app  
*   **База данных:** PostgreSQL (Cloud Supabase Cluster)

## ✨ Ключевой функционал

*   **Приоритезация задач:** Интерактивная установка маркера важности (иконка звезды). Двухуровневая сортировка выполняется на стороне СУБД, автоматически поднимая критические дела наверх.
*   **Клиентская фильтрация:** Моментальное переключение списков по категориям «Все», «Активные» и «Выполненные» без повторных сетевых запросов.
*   **Инструменты REST API:** Полный CRUD-цикл управления сущностями задач (выборка, создание, безопасное обновление статусов без ошибок 500, удаление).
*   **DevTools UI & Адаптивность:** Строгий темный дизайн в стиле утилит разработчика с полноценной адаптивной версткой для мобильных телефонов.

## 📁 Архитектура проекта (MVC Бэкенд)





```text
backend/
├── config/
│   └── db.js          # Изолированный модуль пула подключений к PostgreSQL
├── controllers/
│   └── taskController.js # Серверная бизнес-логика и обработка SQL-запросов
├── routes/
│   └── taskRoutes.js  # Маршрутизация эндпоинтов REST API
├── server.js          # Точка входа, middleware ядра и раздача статики
```

## 🛠 Технологический стек

*   **Backend:** Node.js, Express.js, `pg` (PostgreSQL client)
*   **Database:** PostgreSQL (Supabase Cloud СУБД)
*   **Frontend:** Vanilla JavaScript (ES6+), HTML5, CSS3 (Custom Media Queries)

## 🔧 Локальный запуск и развертывание

### 1. Подготовка окружения
Создайте в корне папки `backend/` файл конфигурации переменных среды `.env`:
```env
PORT=8080
SUPABASE_DB_URL=your_postgresql_connection_string_here
```

### 2. Установка зависимостей и запуск бэкенда
```bash
cd backend
npm install
npm start
```
Сервер запустится локально на порту `8080`. Веб-интерфейс будет доступен по адресу `http://localhost:8080`.
